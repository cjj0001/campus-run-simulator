1 注册 GitHub 账号

1.1 打开 GitHub 官网

在浏览器地址栏输入：https://github.com

📸 截图参考：GitHub 首页

页面右上角有「Sign up」和「Sign in」按钮

页面中央有大大的输入框用于注册

1.2 点击注册按钮

点击页面右上角的 Sign up 按钮（白色文字，黑色背景按钮）

1.3 填写注册信息

① 输入邮箱地址

建议使用常用邮箱（QQ邮箱、163邮箱、Gmail均可）

② 设置密码

密码长度至少 8 位，包含字母和数字

③ 设置用户名

这个用户名会出现在你的网站链接中，建议简洁好记

例如：xiaoming2024、runner-boy、sports-user

④ 邮箱验证

接收验证码邮件，输入页面上显示的验证码

💡 小提示

用户名一旦确定，你的网站地址就是：https://你的用户名.github.io/仓库名/，请慎重选择！

1.4 完成注册

按照页面提示完成人机验证，选择免费的 Free 计划即可。

2 创建新代码仓库

2.1 登录后进入主页

登录成功后，你会看到 GitHub 个人仪表盘页面

2.2 点击创建仓库

方式一：点击左侧边栏的 New 绿色按钮

方式二：点击页面右上角的 + 号，选择「New repository」

📸 截图参考：创建仓库按钮位置

左侧边栏顶部：绿色按钮写着 "New"

右上角：+ 号图标，点击展开菜单

2.3 填写仓库信息（重点！）

设置项 填写内容 说明
Repository name campus-run-simulator 仓库名称，建议用这个
Description 校园跑路径模拟系统 可选，简单描述一下
Public / Private ✅ Public 必须选公开！私有仓库无法用 Pages
Add a README file ⬜ 不勾选 我们自己上传文件
Add .gitignore ⬜ 不勾选 不需要
Choose a license ⬜ 不勾选 可选 MIT 协议

⚠️ 重要提醒

一定要选择 Public（公开）！私有仓库只有付费用户才能使用 GitHub Pages 功能。

2.4 创建仓库

确认所有设置后，点击页面底部的绿色按钮 Create repository

3 网页端上传文件

3.1 进入上传界面

创建仓库成功后，你会看到一个几乎空白的仓库页面

找到并点击：uploading an existing file 链接

📸 截图参考：上传链接位置

在 Quick setup 标题下方，蓝色文字链接

文字内容："...or upload an existing file"

3.2 准备需要上传的文件

请确保你已下载以下文件到你的电脑：

  • 📄 index.html - 主应用页面（必须）
  • 📄 docs.html - 使用文档（推荐）
  • 📄 github-deploy-guide.html - 部署教程（可选）

3.3 拖拽上传文件

  1. 在文件资源管理器中选中上面的文件
  2. 按住鼠标左键，拖拽到浏览器中的上传区域
  3. 松开鼠标，文件会自动开始上传

📸 截图参考：上传区域

页面中央有虚线框，写着 "Drag files here to add them to your repository"

也可以点击 "choose your files" 选择文件

💡 备选方案

如果拖拽不成功，可以点击「choose your files」按钮，在弹出的文件选择窗口中选中文件。

3.4 确认并提交文件

文件上传完成后：

  1. 滚动到页面底部
  2. 在 "Commit changes" 区域保持默认即可
  3. 点击绿色按钮 Commit changes

✅ 验证上传成功

提交后页面会自动刷新，你应该能看到文件列表显示了上传的 index.html 等文件。

4 启用 GitHub Pages

4.1 进入仓库设置

在仓库页面的顶部导航栏，找到并点击 Settings 选项卡（最右侧）

📸 截图参考：导航栏位置

导航栏选项顺序：Code → Issues → Pull requests → Actions → Projects → Security → Insights → Settings

4.2 找到 Pages 设置

在 Settings 页面的左侧菜单中，向下滚动找到：

→ Pages （在 Code and automation 分类下）

4.3 配置 Pages 源（关键步骤！）

① Source（源）设置

下拉框选择：Deploy from a branch

② Branch（分支）设置

第一个下拉框：main 或 master

（一般是 main，旧仓库可能是 master）

第二个下拉框：/ (root)

表示从根目录读取文件

③ 保存设置

点击 Save 按钮

⚠️ 注意

点击 Save 后页面会刷新。此时顶部会显示黄色提示："Your site is ready to be published"

4.4 等待部署完成

GitHub 需要 1-3 分钟来部署你的网站：

  1. 刷新 Pages 设置页面
  2. 顶部提示变成绿色："Your site is live at https://..."
  3. 表示部署成功！

✅ 部署成功标志

绿色提示条 + 可点击的网站链接 = 部署成功！

6 常见问题排查

❌ 问题1：访问网站显示 404 Not Found

可能原因及解决方案：

  1. 部署还没完成 - 等待 3-5 分钟，刷新页面再试
  2. 地址输入错误 - 检查用户名和仓库名拼写是否正确
  3. 文件名错误 - 确保主文件名叫 index.html（全小写）
  4. 分支设置错误 - 回到 Pages 设置，确认选择了 main + /root

❌ 问题2：更新文件后网站不更新

解决方案：

  1. 强制刷新浏览器 - 按 Ctrl + F5（Windows）或 Cmd + Shift + R（Mac）
  2. 清除浏览器缓存 - 或使用无痕模式访问
  3. 等待 GitHub 更新 - 一般需要 1-2 分钟

❌ 问题3：地图不显示 / 显示空白

解决方案：

  1. 检查网络连接 - 地图需要加载在线资源
  2. 等待页面完全加载 - 地图资源较大，加载可能需要几秒
  3. 使用 Chrome / Edge 浏览器 - 兼容性最好

❌ 问题4：找不到 Pages 设置选项

解决方案：

  1. 确认仓库是 Public - 私有仓库免费用户不能用 Pages
  2. 在 Settings 页面向下滚动 - Pages 选项在下方
  3. 确认你是仓库所有者 - 只有所有者能修改 Pages 设置

❌ 问题5：上传文件后不显示

解决方案：

  1. 确认点击了 Commit changes - 必须提交才会真正保存
  2. 刷新 Code 页面 - 按 F5 刷新
  3. 检查文件名 - 不要使用中文文件名或特殊字符

💡 终极解决方法

如果以上方法都无效，删除仓库，按照教程重新来一遍！大多数问题都是某一步设置错误造成的。
