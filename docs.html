🚀 快速开始

方式一：直接运行（最简单）

直接用浏览器打开 index.html 文件即可使用，无需任何后端服务。

方式二：本地服务器（推荐）

# 使用 Python 3 python3 -m http.server 8080 # 使用 Node.js npx serve . # 使用 PHP php -S localhost:8080

然后访问 http://localhost:8080

方式三：在线部署

  • • GitHub Pages：Fork 仓库，在 Settings 中启用 Pages
  • • Vercel / Netlify：导入仓库，一键部署
  • • 任何静态托管：支持纯静态文件托管服务

📖 使用教程

1

添加点位

点击地图任意位置添加打卡点，或点击「加载预设点位」使用操场环形路径。至少需要添加 2 个点位。

2

调整参数

设置目标距离（1-5km）、平均速度（推荐2.5-3.5m/s）、GPS漂移（推荐3-8米）、速度波动（推荐15-25%）。

3

预览路径

点击「生成路径预览」查看规划路线，地图上会显示蓝色预测轨迹。

4

开始模拟

点击「开始模拟」启动实时模拟，可随时暂停/继续。观察距离、速度、时长实时变化。

5

导出数据

模拟完成后可导出 GeoJSON、GPX、JSON 三种格式的轨迹文件。

🧠 核心算法说明

路径平滑算法

使用二阶贝塞尔曲线对原始路径进行平滑处理

lat = (1-t)² * p0.lat + 2*(1-t)*t * midLat + t² * p1.lat

S型速度曲线

  • • 0-10%：起步加速（20%→100%）
  • • 10-90%：匀速（带±波动）
  • • 90-100%：结束减速

GPS漂移模拟

基于高斯噪声模型，考虑纬度修正

latDrift = (meters / 111000) * random lngDrift = meters / (111000 * cos(lat))

数据自洽校验

  • • 时间戳间隔一致性
  • • 距离 = 速度 × 时间
  • • 加速度合理范围

推荐仓库结构

初始化 Git 仓库

# 创建新仓库 git init git add . git commit -m "Initial commit: 校园跑路径模拟系统 v1.0" # 添加远程仓库 git remote add origin https://github.com/你的用户名/campus-run-simulator.git git push -u origin main

启用 GitHub Pages

  1. 进入仓库 Settings 页面
  2. 找到 Pages 选项
  3. Source 选择 main 分支
  4. 点击 Save，等待部署完成
  5. 访问 https://你的用户名.github.io/campus-run-simulator/

🔬 创高体育技术调研

📍 定位机制

  • • 主定位：GPS/GLONASS 卫星定位
  • • 辅助定位：WiFi、基站、蓝牙信标
  • • 定位精度：户外约 5-15 米
  • • 采样频率：约每 5 秒记录一次

🔐 防作弊校验

  • • 速度校验：配速需在 6-10 分钟/公里
  • • 点位校验：必须经过所有打卡点（≤50米）
  • • 轨迹校验：路径连续性、无瞬移
  • • 传感器校验：加速度计、陀螺仪数据交叉验证

✅ 有效条件

  • • 完成规定距离（通常 1-2 公里）
  • • 配速在合理范围内
  • • 经过所有强制打卡点位
  • • 运动轨迹无明显异常

📤 导出格式说明

GeoJSON 格式

标准地理数据格式，可导入 QGIS、ArcGIS 等 GIS 软件进行数据分析。

GPX 格式

GPS 交换格式，可导入 Garmin、咕咚、Keep 等运动设备和跑步 APP。

JSON 格式

原始数据格式，包含完整的时间戳、坐标、速度、距离信息，便于二次开发。
